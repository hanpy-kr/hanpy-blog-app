# Redie cli

Redis는 빠른 인메모리 데이터 저장소로, 캐싱, 세션 관리, 메시지 브로커 등 다양한 용도로 널리 사용됩니다. 이 글에서는 Mac 환경에서 Redis CLI를 설치하는 방법과 몇 가지 기본 명령어를 통해 Redis 서버와 상호작용하는 방법을 알아봅시다.

---

## 1. Mac에서 Redis CLI 설치하기

Mac에서는 Homebrew를 사용하여 Redis를 쉽게 설치할 수 있습니다. Homebrew는 macOS에서 패키지 관리를 간편하게 도와주는 도구입니다. 터미널을 열고 아래의 명령어를 입력하여 설치를 진행합니다.

```bash title="cmd" showLineNumbers
➜ brew install redis
```

이 명령어는 Redis 서버와 함께 기본적으로 Redis CLI(redis-cli)를 설치합니다. 설치가 완료되면, 터미널에서 redis-cli 명령어를 통해 CLI에 접근할 수 있습니다.

## 2. Redis CLI 실행 및 기본 명령어

Redis CLI를 실행하려면 터미널에 다음 명령어를 입력합니다.

```bash title="cmd" showLineNumbers
➜ redis-cli
```

CLI 프롬프트가 나타나면, 여러 가지 Redis 명령어를 사용할 수 있습니다. 몇 가지 기본 명령어를 살펴보겠습니다.

### 2-1. PING

Redis 서버와의 연결을 확인하기 위해 사용합니다.

```bash title="cmd" showLineNumbers
➜ redis-cli ping
```

### 2-2. SET & GET

데이터를 저장하고 조회할 때 사용하는 기본적인 키-값 명령어입니다.

```bash title="cmd" showLineNumbers
➜ redis-cli set mykey "Hello, Redis!"
➜ redis-cli get mykey
```

위 명령어들은 mykey라는 키에 "Hello, Redis!"라는 값을 저장하고, 그 값을 조회할 수 있습니다.

```bash title="cmd" showLineNumbers
➜ redis-cli keys *
```

위 명령어는 저장된 모든 키를 반환합니다. 하지만, 많은 key가 존재하면 성능에 악영향을 줄 수 있기 때문에 운영 환경에서는 사용을 지양하여야 합니다. 운영환경에서는 아래와 같이 scan 명령어를 사용하는것이 권장됩니다.

```bash title="cmd" showLineNumbers
➜ redis-cli scan 0
```

SCAN 명령어는 반복 호출을 통해 점진적으로 키 목록을 검색하므로, 시스템에 부하를 주지 않고 키 목록을 조회할 수 있는 장점이 있습니다.

### 2-3. DEL

특정 키에 저장된 데이터를 삭제합니다.

```bash title="cmd" showLineNumbers
➜ redis-cli del mykey
```

이 명령어로 mykey에 해당하는 데이터를 삭제할 수 있습니다.

### 2-3. INFO

Redis 서버의 상태와 다양한 정보를 확인할 수 있습니다.

```bash title="cmd" showLineNumbers
➜ redis-cli info
```

서버의 메모리 사용량, 연결 정보, 버전 정보 등 여러 유용한 데이터를 출력합니다.

---

Mac 환경에서 Homebrew를 사용하여 Redis를 설치하면, 자동으로 Redis CLI도 함께 설치되어 손쉽게 Redis 서버와 상호작용할 수 있습니다. 기본적인 명령어들을 통해 Redis 서버의 상태를 확인하고, 데이터를 저장 및 삭제하며, 개발 및 테스트 환경에서 Redis의 다양한 기능을 빠르게 활용할 수 있습니다. 이 글을 통해 Mac에서 Redis CLI를 설치하고 사용하는 방법을 간단히 알아보았습니다. Redis의 더욱 다양한 기능과 활용법에 대해서는 알아보도록 합시다.

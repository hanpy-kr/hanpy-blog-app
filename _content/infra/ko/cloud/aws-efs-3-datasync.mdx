# AWS EFS - DataSync

AWS EFS(Elastic File System)는 다중 AZ 기반의 고가용 파일 시스템이지만, 백업 기능을 기본으로 제공하지 않기 때문에 별도의 데이터 보호 전략이 필요합니다. 이때 AWS Backup이 관리형 백업에 적합하다면, AWS DataSync는 EFS 데이터를 다른 스토리지로 복사하는 데 특화된 서비스입니다. DataSync는 백업보다는 데이터 이동, 동기화, 장기 보관에 초점을 둔 서비스로, EFS 데이터를 S3, 다른 EFS, 온프레미스 환경으로 안전하고 빠르게 전송할 수 있습니다.

---

## AWS DataSync를 사용하는 방식

AWS DataSync는 대용량 데이터 전송을 위해 설계된 관리형 데이터 이동 서비스입니다. EFS를 소스(Source) 또는 타겟(Target)으로 사용할 수 있으며, 파일 단위로 데이터를 복사합니다.

별도의 복잡한 네트워크 구성 없이 AWS에서 관리형으로 제공되며, 스케줄 기반 실행과 증분 복사가 가능합니다.

- EFS ↔ S3 데이터 전송 지원
- EFS ↔ 다른 EFS 간 데이터 복사
- EFS ↔ 온프레미스 환경 연동 가능
- 파일 메타데이터(권한, 타임스탬프) 유지
- 증분 복사 지원

---

## EFS와 DataSync의 구조적 특성

EFS는 블록 스토리지가 아닌 파일 시스템이기 때문에 스냅샷 개념이 없으며, DataSync 역시 파일 단위 복사 방식으로 동작합니다. 이로 인해 특정 시점의 상태를 완전히 보장하는 백업보다는, 데이터를 다른 위치에 안전하게 복사해 두는 개념으로 이해하는 것이 적절합니다.

---

## 비용 관점에서 최소한 알아야 할 점

### AWS DataSync 비용

* 전송된 데이터 양 기준 과금
* 초기 전체 복사 시 비용 부담 큼
* 이후 변경 데이터만 증분 전송

### 목적에 따른 비용 고려

* 장기 보관 시 S3 스토리지 비용 발생
* 크로스 리전 전송 시 추가 네트워크 비용 발생 가능
* 온프레미스 연계 시 네트워크 구성 비용 고려 필요

DataSync는 전송 빈도와 데이터 크기가 비용에 가장 큰 영향을 미치므로, 운영 목적에 맞는 주기 설정이 중요합니다.

---

## 정리

EFS는 고가용성을 제공하지만 백업 기능을 기본으로 포함하지 않으며, DataSync는 이러한 EFS 데이터를 다른 스토리지로 안전하게 복사하는 데 적합한 서비스입니다. 관리형 백업과 복구가 목적이라면 AWS Backup이 적합하고, 데이터 이동, 장기 보관, 리전 간 복제가 목적이라면 AWS DataSync가 효과적인 선택이 됩니다. 또한 EFS와 DataSync 모두 파일 단위로 동작하며 스냅샷 개념이 없다는 점을 이해하고 사용하는 것이 중요합니다.
